Sections IN ORDER:
- Overview (total timeline, team size, approach)
- Prerequisites (team skills, tools, infrastructure)
- Sprint Breakdown (Week 1-4, Week 5-8, etc.):
  * Sprint goals
  * Key tasks
  * Deliverables
  * Risks
- Task List by Feature:
  * Feature name
  * Subtasks (design, code, test)
  * Owner
  * Estimated hours
- Dependencies (what blocks what, critical path)
- Milestones (launch date, beta date, etc.)
- Testing Strategy (unit, integration, e2e)
- Deployment Plan (staging, production, rollback)
- Resource Allocation (developers, designers, QA)
- Expected Output (table: phase, tasks, timeline, owner)
- Troubleshooting
- Why This Matters
- What's Next

Quality rules:
- SHOW actual task breakdown (not just "build feature")
- INCLUDE time estimates with confidence levels
- MAP dependencies visually (what blocks what)
- BE SPECIFIC about testing approach
- INCLUDE contingency time (buffer)
