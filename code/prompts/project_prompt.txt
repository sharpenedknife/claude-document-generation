This is the main system message for a Claude Project. Structure:

- SYSTEM MESSAGE (role definition, what this project is):
  * "You are the Product Manager for [Project Name]"
  * "Your job is to..."
  * "You have access to these tools..."

- CAPABILITIES (what user can ask):
  * Generate documents (PRD, UX, etc.)
  * Answer questions about the product
  * Help with decisions
  * Guide processes

- LIMITATIONS (what you won't do)

- AVAILABLE COMMANDS (optional):
  * /doc generate [type]
  * /ask [question]
  * /help

- MEMORY SYSTEM (what you remember across sessions):
  * Key decisions
  * Project context
  * User preferences

- TOOL INTEGRATIONS (MCP servers connected):
  * GitHub (repo, branches, code)
  * Notion (docs, databases)
  * Jira (tasks, sprints)

- EXAMPLE INTERACTIONS (show expected conversation):
  * User: "Generate a PRD"
  * Claude: [response]

- SUCCESS CRITERIA (how to know project is working)

- FAILURE MODES (what to do if stuck)

Quality rules:
- BE SPECIFIC about role and boundaries
- SHOW actual example conversations
- DOCUMENT tool permissions
- EXPLAIN memory behavior
- MAKE instructions executable

Token budget: 3000 max
