# Research Hub — Session Starter
**Copy, fill in the blanks, paste into Research Hub to start a research session.**

---

## Context Card

```
PROJECT: [what you're building / working on — 1 sentence]
STAGE:   [e.g. "pre-idea validation" / "post-MVP, planning v2" / "deciding tech stack"]
GOAL:    [what decision this research needs to support — 1 sentence]
URGENCY: [e.g. "need output today" / "thorough is more important than fast"]
```

---

## Research Mode — Pick One and Paste the Block

---

### MODE 1 — Competitive Research

```
RESEARCH MODE: Competitive

QUESTION: Who are my competitors and where is the gap I can win?

PRODUCT: [what it does + who it's for — 2 sentences]
MARKET:  [geography / industry / customer segment]
FOCUS:   [e.g. "pricing model", "underserved features", "positioning"]

OUTPUT FORMAT:
- Competitor table: Name | Core features | Weakness | Pricing
- Differentiation gap: top 3 opportunities
- Recommended positioning: 1–2 sentences
- Sources used
```

---

### MODE 2 — Market Sizing

```
RESEARCH MODE: Market Sizing

QUESTION: How big is this opportunity and is it worth pursuing?

PRODUCT: [what it does + who it's for]
CUSTOMER SEGMENT: [who specifically — role, company size, geography]
TIMEFRAME: [current market / 3-year / 5-year]

OUTPUT FORMAT:
- TAM / SAM / SOM with method (top-down or bottom-up)
- Key assumptions (mark each CONFIRMED / INFERRED / ASSUMED)
- Sources used
- Confidence level: High / Medium / Low
```

---

### MODE 3 — Technical Deep-Dive

```
RESEARCH MODE: Technical

QUESTION: [what you need to decide — e.g. "which vector DB fits my use case"]

CONTEXT: [what you're building, scale, team skill level]
OPTIONS: [technologies or approaches to compare — list them]
DECISION CRITERIA: [what matters most — e.g. latency, cost, developer experience]

OUTPUT FORMAT:
- Comparison table: Option | Pros | Cons | Best for | Cost
- Recommendation with rationale
- Gotchas / things to watch out for
- Sources used
```

---

### MODE 4 — User / Customer Research

```
RESEARCH MODE: User Research

QUESTION: [what you want to understand about users — e.g. "what's their current workflow"]

TARGET USER: [role, context, technical level]
PROBLEM AREA: [what problem your product solves for them]
SOURCES: [e.g. "Reddit, G2 reviews, Twitter/X, your own interviews"]

OUTPUT FORMAT:
- Current workflow (step-by-step, without my product)
- Top 3 pain points with severity
- First value moment: what makes them say "this works"
- Common objections
- Direct quotes if found
```

---

### MODE 5 — Document Analysis

```
RESEARCH MODE: Document Analysis

QUESTION: [what you want to extract or understand]

[Attach or paste the document(s) here]

OUTPUT FORMAT:
- Key findings: numbered, each with source location
- Answer to my question directly
- Anything contradicted or uncertain
- Gaps: what the document doesn't cover
```

---

### MODE 6 — Synthesis

```
RESEARCH MODE: Synthesis

QUESTION: What's the single coherent picture from all these findings?

INPUTS: [paste or describe the research you've already collected]
OUTPUT FOR: [where this goes — e.g. "PRD", "investor pitch", "architecture decision"]

OUTPUT FORMAT:
- Executive summary: 3 sentences
- Key insights: numbered, each tagged CONFIRMED / INFERRED / ASSUMED
- Recommendation / next action
- Open questions that still need answers
```
