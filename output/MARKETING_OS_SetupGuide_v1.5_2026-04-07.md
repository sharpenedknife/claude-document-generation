# Marketing Hub — Setup Guide
**Version:** 1.5 | **Date:** 2026-04-07
**Architecture:** Claude = Brain + HTML Production | ChatGPT = Research + First Drafts
**New in v1.5:** Dual Claude config — Chat (claude.ai Projects) + Cowork (desktop app)

---

## Choose Your Claude Interface

| You're doing | Use |
|---|---|
| Quick campaign planning, on mobile, sharing with clients/team | **Claude.ai Chat** |
| Heavy sessions, file management, skill invocation, HTML to disk | **Cowork** |
| Both | Set up both — they complement each other |

## PART 1 — CLAUDE.AI CHAT SETUP

*Full-featured for anywhere-access, mobile, team sharing, and shareable artifact links.*

### Feature map

| Feature | Where | What it does |
|---|---|---|
| Project | Projects → New | Persistent workspace, 200K context |
| Knowledge files | Project → Knowledge | Context loaded into every conversation |
| Custom Instructions | Project → Settings | Claude's role, behavior, output rules |
| Conversation Starters | Project → Settings | 4 clickable entry buttons |
| Custom Style | Profile → Styles | Response format baked in — no re-prompting |
| Memory | Automatic | ICP, campaigns, state across sessions |
| Artifacts | Automatic | HTML or Markdown in side panel — shareable link |

---

### Step 1: Create the Project

**claude.ai → Projects → New Project**
- Name: `Marketing OS`
- Description: `Marketing brain — strategy, briefs, HTML production, evaluation`

---

### Step 2: Upload Knowledge Files

**Project → Knowledge → Add content → Upload files**

Required:
```
MARKETING_OS_CLAUDE_v1.6_2026-04-07.md   ← Claude's instructions
```

When you have Documentation Builder output for a product:
```
CONTEXT_Product.md    ← generated by Documentation Builder
CONTEXT_Technical.md  ← generated by Documentation Builder
```

Until then: tell Claude in your first session — "Remember: Product = [X], ICP = [Y]" — it stores in memory.

---

### Step 3: Custom Instructions

**Project → Settings → Custom Instructions** — paste:

```
You are Marketing OS — the strategic brain and HTML production engine of my marketing system.

ARTIFACT RULE: Open artifact for ANY output ≥10 lines.
Briefs → Markdown artifact. HTML pages → HTML artifact. Evaluations → Markdown artifact.

DO NOT write raw copy — generate briefs for ChatGPT GPTs.
EXCEPTION: Denis pastes a ChatGPT draft + asks to upgrade → rewrite in Markdown artifact.
HTML PAGES: "Build me a landing page" / "production page" → build complete responsive HTML artifact.

See MARKETING_OS_CLAUDE_v1.6 in Project Knowledge for full routing, brief format, and evaluation format.
```

```

```

### Step 5: Custom Style

**Profile → Styles → Create Style** — name: `Marketing OS`

Paste into instructions:
```
Direct, no preamble. Lead with the action or answer.
Never: "Great question", "Absolutely", "Certainly".
Output ≥10 lines → open artifact (HTML or Markdown).
Briefs: use 📋 BRIEF format with dividers.
Evaluations: use 📊 EVALUATION format, quote failing lines.
HTML pages: inline CSS, responsive, self-contained, commented sections.
Tone: confident, precise, marketing-literate.
```

Activate this style inside the Marketing OS project.

### Step 6: Memory

## PART 2 — COWORK SETUP

*For heavy sessions: file management, direct skill invocation, HTML written to disk.*

### Feature map

| Feature | How it works in Cowork |
|---|---|
| CLAUDE.md | Auto-loaded from workspace root on every session |
| Context files | Read from disk: context/PRODUCT.md · ICP.md · CAMPAIGNS.md · MESSAGING.md |
| Skills | Invoked directly with Skill tool (`/setup`, `/brief`, `/evaluate`) |
| Memory | Auto-memory system (MEMORY.md + individual memory files) |
| Outputs | Written to disk (output/pages/ · output/copy/ · output/strategy/) |
| File sharing | present_files tool — Denis gets clickable file card |
| Task tracking | TodoWrite — multi-step workflow progress visible in UI |

---

### Step 1: Mount Your Marketing OS Folder

Open Cowork desktop app → select your `Marketing OS` folder as the workspace.

The folder should have this structure (from the `cowork/` directory in this bundle):
```
Marketing OS/
├── CLAUDE.md                    ← auto-loaded on session start
├── context/
│   ├── PRODUCT.md               ← fill this in first
│   ├── ICP.md                   ← updated after ICP research runs
│   ├── CAMPAIGNS.md             ← updated after every campaign session
│   └── MESSAGING.md             ← updated after positioning sessions
├── briefs/                      ← Claude saves briefs here
├── output/
│   ├── pages/                   ← production HTML pages
│   ├── copy/                    ← evaluated copy
│   └── strategy/                ← GTM + strategy docs
└── evaluation/
    └── LOG.md                   ← automatic evaluation log
```

---

### Step 2: Fill In context/PRODUCT.md

Open `context/PRODUCT.md` and fill in every section before your first session.
This is the only required manual step. Claude reads it on every session start.

If you've already run Documentation Builder for this product: paste the relevant sections from `CONTEXT_Product.md` into the context files — they map directly.

---

### Step 3: First Session — Run /setup

Open Cowork → type `/setup`

Claude will:
1. Invoke the `product-marketing-context` skill
2. Ask guided questions about your product + ICP
3. Save the output to `context/` files
4. Confirm what it remembered

This populates your context files if you haven't filled them in manually.

---

### Step 4: Slash Commands

| Command | What happens |
|---|---|
| `/setup` | Invoke `product-marketing-context` skill, populate context files |
| `/brief [gpt] [goal]` | Generate brief immediately, save to briefs/ |
| `/evaluate` | Claude asks for ChatGPT output, evaluates, logs to evaluation/LOG.md |
| `/build [page name]` | Build HTML landing page, write to output/pages/, present_files |
| `/status` | Read CAMPAIGNS.md, show active campaigns + last scores |
| `/skills` | Show Tier 2 skill routing table |

---

### Step 5: Skill Invocation

In Cowork, Claude can invoke skills directly. Denis doesn't need to tell Claude which skill — Claude routes based on the request. But Denis can also invoke explicitly:

```
"Use the cold-email skill to write this sequence"
"Run competitive-analysis on these three competitors"
"Invoke customer-research on these interview transcripts" [attach files]
"Run ab-test-setup for this campaign"
```

Skill escalation is automatic: if a ChatGPT GPT output scores <5 after revision, Claude invokes the equivalent skill without Denis having to ask.

---

### Cowork use cases

- Heavy ICP research: upload interview transcripts, Claude runs `customer-research` skill
- Full competitive analysis: Claude runs `competitive-analysis`, saves to output/strategy/
- Production HTML pages: written to disk, shared via present_files
- Content calendar: `content-strategy` skill → saves to output/strategy/
- A/B test design: `ab-test-setup` skill → saves to output/strategy/
- Multi-step campaigns: TodoWrite tracks every step, Claude updates CAMPAIGNS.md as it goes
- Evaluation log: every score is written to evaluation/LOG.md automatically

## PART 4 — DAILY WORKFLOWS

### Workflow: Chat — Quick Loop
```
1. claude.ai → Marketing OS → click 🧠 or type request
2. Claude → brief in Markdown artifact → copy it
3. Paste in ChatGPT GPT → canvas output
4. Copy from canvas → paste in Claude → "Evaluate this"
5. Claude → 📊 evaluation artifact → SHIP or revision brief
```

### Workflow: Chat — Build Production Page
```
1. claude.ai → Marketing OS → click 🏗️ Build me a landing page
2. Claude asks: product, ICP, CTA, proof
3. Claude builds HTML artifact
4. Share artifact link with client or dev → done
```
*Or: run Landing Builder GPT for draft first, paste back to Claude, Claude builds HTML from the draft.*

### Workflow: Cowork — Heavy Research Session
```
1. Open Cowork → Marketing OS folder
2. Claude reads context/ files automatically
3. Upload customer interview transcripts
4. /brief ICP or: "Run customer-research on these transcripts"
5. Claude invokes skill → saves output to output/strategy/
6. Claude updates context/ICP.md with new profiles
7. Claude updates CAMPAIGNS.md with session summary
```

### Workflow: Cowork — Full Campaign Build
```
1. /brief Content Builder [goal]
2. Claude generates brief → saves to briefs/
3. Denis pastes brief in ChatGPT Content Builder
4. Denis pastes output back in Cowork
5. /evaluate → Claude scores, logs to evaluation/LOG.md
6. ≥7: Claude saves to output/copy/, presents file
7. <7: Claude writes revision brief → saves to briefs/ → repeat
```
